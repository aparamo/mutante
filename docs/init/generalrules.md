## General rules

### UI / design
- Mobile first, but a great specific experience for each, for small and big screens
- Maintain clean style and maintain a consistent style. 
- Always use shadcn for UI basic components

## Always take into account

- React, NextJS and web dev BEST PRACTICES (React patterns, encapsulation, one function per component, etc)
- Follow the current style, reuse what we already have, follow the same patterns and structure
- React and it's important patterns.
- Type safety is one of the most important requisites, also as part of best practices, always define types, interfaces, props, typefiles, use payload types, zod and all available tools, each in their right context. Avoid "as any" types or es-lint disables or error hiding, those are not solutions and are very bad practice. Provide excelent quality code.
- We use bun (bun run dev, bun run build, etc) not npm or pnpm
- Maintain a general perspective wide enough so you fully understand how to effectively and efficiently continue adding or making changes to the project, without breaking things and really solving the issues.
